================================================================================
    📊 CARA MEMBACA HASIL TEST
================================================================================

Setelah test selesai, Anda akan lihat output seperti ini:

✓ ✅ Homepage loaded................: 98.5%
✓ ⚡ Response < 2s..................: 97.2%
✗ ✅ API responds...................: 89.3%  ← Ada yang failed

http_req_duration...: avg=456ms min=123ms max=2.1s p(95)=1.2s
http_req_failed.....: 1.8% (23 failed / 1280 total)
http_reqs...........: 1280 (32/s)
vus_max.............: 10


INTERPRETASI:
────────────────────────────────────────────────────────────────────────────

1. Error Rate (http_req_failed)
   ✅ < 1%        = Server sangat kuat
   ⚠️ 1-5%       = Mendekati limit
   🔴 > 5%        = Server kewalahan (INI BATAS MAKSIMAL!)

2. Response Time (p95)
   ✅ < 500ms     = Sangat cepat
   ⚠️ < 1500ms    = Masih OK
   🔴 > 2000ms    = Terlalu lambat

3. Max Users (vus_max)
   Ini adalah jumlah concurrent users saat test


CONTOH:
────────────────────────────────────────────────────────────────────────────

Hasil Test A:
  Error rate: 0.5%
  p(95): 456ms
  vus_max: 200
→ ARTINYA: Server kuat sampai 200 users! ✅

Hasil Test B:
  Error rate: 15.3%
  p(95): 3.2s
  vus_max: 100
→ ARTINYA: Server tidak kuat di 100 users! Batas maksimal < 100 🔴
